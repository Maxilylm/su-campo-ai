Architecture Approaches

How to evolve CampoAI from a livestock MVP to a configurable farm management platform

A

Modular Tabs — Extend the Monolith

Keep the current single-page architecture but extract each tab into its own component file. Add 4 new tab components (Agricultura, Inventario, Finanzas, Métricas). Farm profile determines which tabs appear.

Pros

  • Least disruption to existing code
  • Fast to implement — just add components
  • Single dashboard, familiar UX

Cons

  • 10 tabs in one dashboard gets crowded
  • page.tsx is already 19KB — adding more makes it harder to maintain
  • All data loads on dashboard init (performance concern with 4 more domains)
B

Grouped Navigation — Section-Based Layout

Group the 10 features into 3-4 top-level sections, each with sub-tabs. E.g., "Producción" (Hacienda + Agricultura + Sanidad), "Gestión" (Inventario + Finanzas), "Análisis" (Métricas + Registro), plus Mapa and Chat. Each section is a separate page with lazy loading.

Pros

  • Scales cleanly — no tab overflow
  • Lazy loading per section = better performance
  • Logical grouping helps farmers find features
  • Each section loads only its own data

Cons

  • Bigger refactor — need to break up the monolith page.tsx
  • Navigation redesign required (sidebar or top nav)
  • More complex routing
C

Hybrid — Smart Sidebar + Adaptive Dashboard

Add a collapsible sidebar for navigation between sections, keep a home dashboard as the landing page (the current Overview tab, enhanced with metrics), and each feature gets its own route. The sidebar adapts to the farm's operation type — hides irrelevant sections. Mobile: sidebar becomes a bottom sheet or hamburger menu.

Pros

  • Professional feel — standard for management apps
  • Infinite scalability — add sections without UI crowding
  • Per-route code splitting = best performance
  • Sidebar visually communicates "this is a serious tool"

Cons

  • Largest refactor — full navigation rewrite
  • Sidebar takes horizontal space on mobile
  • More complex state management across routes

💡 My Recommendation: B (Grouped Navigation)

Option B hits the sweet spot. It's a meaningful upgrade from the tab monolith without the full complexity of a sidebar app. The grouped sections (Producción, Gestión, Análisis) create a natural mental model for farmers, and lazy loading per section keeps performance tight. The refactor of page.tsx is actually needed regardless — at 19KB it's already overdue for splitting.
