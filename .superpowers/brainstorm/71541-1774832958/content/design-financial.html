Design Section 4: Finanzas (Cost-Per-Unit Tracking)

Where the money goes — profitability per herd, per crop, per section

Data Model

CREATE TABLE financial_transactions (
  id UUID PRIMARY KEY,
  farm_id UUID REFERENCES farms ON DELETE CASCADE,
  type TEXT NOT NULL, -- ingreso, egreso
  category TEXT NOT NULL, -- venta_ganado, venta_cosecha, compra_insumo, servicio, mano_obra, transporte, veterinario, maquinaria, otro
  description TEXT,
  amount NUMERIC NOT NULL, -- always positive; type determines sign
  currency TEXT DEFAULT 'USD', -- USD, UYU, ARS
  date DATE NOT NULL DEFAULT CURRENT_DATE,
  -- Cost allocation (at least one should be set for egresos)
  section_id UUID REFERENCES sections,
  crop_id UUID REFERENCES crops,
  cattle_id UUID REFERENCES cattle,
  -- Link to inventory if this was a purchase
  inventory_movement_id UUID REFERENCES inventory_movements,
  notes TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- No separate "accounts" or "ledgers" — one flat table.
-- Cost-per-unit is computed at query time via aggregation:
-- Cost per head = SUM(egresos WHERE cattle_id = X) / cattle.count
-- Cost per hectare = SUM(egresos WHERE crop_id = X) / crop.planted_hectares

How Costs Flow In

📦 Inventario
→ Every compra movement auto-creates an egreso transaction, linked to the movement. Every uso movement allocates existing cost to section/crop/cattle.
💉 Sanidad
→ Veterinary visits, treatments can be logged as egreso (category: veterinario), allocated to section/herd.
🎙️ Chat AI
→ Direct voice entry: "Pagué $50,000 por el flete de la hacienda" → egreso, category: transporte
📝 Manual
→ Dashboard form for transactions that don't come from other sections (labor, machinery rental, etc.)

AI Chat Examples

🎙️ "Vendí 10 novillos a 4,250 dólares cada uno"
→ Creates ingreso (venta_ganado), amount: $42,500
🎙️ "Pagué 50 mil pesos de flete para mover la hacienda"
→ Creates egreso (transporte), amount: $50,000 UYU
🎙️ "¿Cuánto me está costando cada cabeza del lote del Norte?"
→ Query: aggregates all egresos allocated to that cattle batch, divides by head count
🎙️ "¿Cuánto gasté este mes?"
→ Query: SUM(amount) WHERE type='egreso' AND date >= first of month

🔗 Key Design Decision: One Table, Not Two

Income and expenses live in the same financial_transactions table differentiated by type. This makes balance calculations trivial (SUM(CASE WHEN type='ingreso' THEN amount ELSE -amount END)) and avoids the complexity of a double-entry accounting system. For cost-per-unit tracking, this is the right level — we're not building QuickBooks.
