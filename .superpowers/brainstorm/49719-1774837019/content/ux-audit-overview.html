CampoAI UX Audit: Current State vs. Professional Standard

Full critique organized by severity. Click to mark issues you agree with.

Critical UX Issues (Must Fix)

1

Hover-triggered navigation dropdowns

Current: onMouseEnter/onMouseLeave on Produccion/Gestion menus. Disappears when cursor drifts off. Unusable on touch devices.

Professional standard: Click-to-open with click-outside-to-close. Linear, Vercel, Stripe all use click-triggered menus. Hover menus are an anti-pattern for navigation.

2

Emoji as UI iconography

Current: 40+ emojis used as icons: 🐄 🌾 💉 🏥 📦 💰 📊 📋 🐣 💀 🤒 etc. Renders differently across OS/browser. Looks unserious.

Professional standard: Consistent icon set (Lucide, Heroicons, or Phosphor). Monochrome, weight-matched, renders identically everywhere. Emoji reserved for user-facing content (chat), never for UI chrome.

3

Inline collapsible forms push content around

Current: Every "Add" action inserts a form inline that shifts the entire page layout. User loses position. 8+ different form patterns, all inline.

Professional standard: Slide-over panels (drawers) or modals for data entry. Page stays stable. Linear uses right-side panels; Stripe uses modals. Content underneath doesn't move.

4

Destructive actions with no confirmation

Current: "Eliminar" buttons directly call deleteSection(), deleteCattle(), etc. No confirmation dialog. One mis-tap deletes data permanently.

Professional standard: Confirmation dialog for all destructive actions. "Are you sure? This will delete X and cannot be undone." Red-themed confirm button.

5

Loading states are emoji + animate-pulse text

Current: Loading shows a giant emoji (🌿, 📊, 💬) with pulsing "Cargando..." text. Feels amateur.

Professional standard: Skeleton screens that mirror the actual layout. Or a minimal spinner with no emoji. Vercel uses skeleton placeholders; Linear uses a subtle progress bar.

Major UX Issues (Should Fix)

6

Select component is redefined in every page

Current: Identical Select component copy-pasted in hacienda, sanidad, agricultura, inventario, finanzas. Each one is the same 15 lines. Browser-native <select> styling is inconsistent across platforms.

Professional standard: One custom Select component with search/filter (like Radix or cmdk). Professional apps never use native <select> for more than 5 options.

7

StatCard is visually flat and information-sparse

Current: Centered text with colored border. No trend indicator, no comparison, no sparkline. Just a big number and a label.

Professional standard: Left-aligned. Small icon. Value + delta/trend badge ("12% vs last month"). Optional sparkline. Stripe Dashboard is the gold standard here.

8

No page headers or breadcrumbs

Current: Pages jump straight into content. No title, no description, no breadcrumb. User navigates to /produccion/sanidad and sees "Vacunaciones" section directly.

Professional standard: Every page has a header with title, optional description, and primary action button. Breadcrumbs for nested pages. Vercel pattern: "Deployments" + filter bar + "Deploy" button.

9

Tables lack professional data table features

Current: Raw HTML tables with no sorting, no pagination, no column resizing, no row selection. Action buttons are tiny "Editar"/"Eliminar" text links.

Professional standard: Sortable columns, pagination for 20+ rows, bulk actions via row selection, hover-revealed action menus (or right-click context menu). Think Linear's issue list.

10

Color system is inconsistent and overloaded

Current: Emerald for primary, success, active nav, AND section colors. Red for errors AND expenses. Amber for warnings AND financial results. Same color means different things in different contexts.

Professional standard: Strict semantic color mapping. One primary action color. Separate semantic palette (success/warning/error). Data visualization uses a different palette entirely.

11

Mobile bottom nav is too simple for 11+ pages

Current: 5 emoji-labeled buttons. "Produccion" goes to hacienda directly, no way to reach sanidad or agricultura from mobile nav. "Gestion" goes to inventario only.

Professional standard: Bottom nav with the 5 icons is fine, but tapping "Produccion" should show a sub-nav sheet or the SubTabBar should be visible on mobile pages. Users need to reach all sub-pages.

Polish Issues (Nice to Fix)

12

Typography hierarchy is too flat

Current: Nearly everything is text-sm (14px). Headers are text-xl (20px). Section headers are text-xs uppercase. Three sizes total across the entire app.

Professional standard: 5-6 size stops with clear hierarchy. Page title (24-28px), section title (18-20px), card title (16px), body (14px), caption (12px). Font weight variation: regular, medium, semibold.

13

No transitions or micro-animations

Current: Forms appear/disappear instantly (conditional rendering). No enter/exit animations. Dropdowns pop in with no transition. Page navigation has no loading indicator.

Professional standard: Subtle fade/slide for panels. Skeleton-to-content transitions. Dropdown menus with 150ms ease-out. Route change indicator (top progress bar like NProgress).

14

Empty states are text-only with emoji

Current: Big emoji + one line of text. No illustration, no action button, no help link.

Professional standard: Illustrated empty state with a clear CTA button. "No cattle registered yet" + illustration + "Add your first cattle" button. Linear's empty states are a good reference.

15

Spacing is too tight throughout

Current: gap-2, gap-3 everywhere. Cards have p-3 or p-4. Everything feels dense and cramped. Tags are tiny (px-2 py-0.5).

Professional standard: More generous padding. Cards with p-5 or p-6. gap-4 between sections. Breathing room signals quality. Luxury = space.

16

Login page feels like a hackathon project

Current: Giant 🐄 emoji as logo. "CampoAI" in emerald. Simple form box. No branding, no value prop, no visual identity.

Professional standard: Split layout (form left, hero/illustration right). SVG logo mark. Tagline with value proposition. Subtle background pattern or gradient. Trust signals.

17

No success/error feedback after actions

Current: After saving a section, cattle, transaction — the form just closes. No toast, no notification, no visual confirmation that it worked. Only failure uses alert().

Professional standard: Toast notifications for all CRUD operations. "Section created successfully" with undo option. Error toasts with retry. Sonner or react-hot-toast.
